# 运行环境工具适配

读取时机：SKILL.md 里的工具名和当前运行环境不一致时。

这个 skill 描述的是能力流程，不绑定单一 agent runtime。执行时按当前环境选择等价工具。

## 本地文件读取

- Codex：优先用 shell 工具读取、转换和检查文件；PDF 可用 `pdftotext`、`pdfinfo`、`pdftoppm` 等本机工具。
- Claude Code：可用 Read 工具直接读文本文件；PDF 过长时分段处理或先抽取纯文本。
- Cursor 或其他 IDE agent：使用内置文件读取能力，必要时通过终端转换为纯文本。

## 网页读取

- 有 WebFetch 能力时，先要求“提取文章全文，保留段落结构和小节标题，不要摘要，不要省略”。
- WebFetch 返回摘要、缺少代码块、分页不完整或官方文档被压缩时，切到浏览器自动化，分段读取 `document.body.innerText`。
- 对官方文档、指南、技术规范和 API 页面，完整性优先于速度。示例、代码块、参数和限制条件不能丢。

## 用户确认

- 有结构化提问工具时，用它让用户选择标题或处理无字幕视频。
- 没有结构化提问工具时，用简短自然语言提问，不要把未确认标题写入 `article.md`。

## 浏览器和截图

- 有 Chrome DevTools MCP 时，按 `visual-capture.md` 的 DOM video 元素截图流程。
- 有 Playwright 或其他浏览器工具时，用等价能力打开页面、定位视频时间、截图视频区域。
- 没有浏览器自动化时，优先抓 YouTube 封面图；正文必须截图但做不到时，保留 `[图:MM:SS - 描述]` 并明确说明未替换。

## 网络和外部命令

- 网络受限时，先使用用户提供的文件、已有字幕或本地缓存。
- 需要下载字幕、拉取网页或安装依赖时，遵守当前 runtime 的授权机制。
- 工具失败后不要无限重试。按 `youtube-transcript-fallback.md` 的失败类型切换路径。
